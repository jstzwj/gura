# Gura

一种基于区域内存管理和无数据竞争并发的系统编程语言。

> **状态：** 早期开发 / 研究原型。

## 概述

Gura 的核心设计原则简洁而强大：**每个可变对象属于一个隔离区域**。这种区域所有权模型无需垃圾回收器即可提供强内存安全保证，同时通过区域转移实现零拷贝并发。

### 主要特性

- **基于区域的内存管理** — 对象按区域分组，每个区域具有明确的生命周期（活跃 → 暂停 → 关闭）
- **无数据竞争的并发** — 可变对象同一时间只能被一个线程访问；共享对象必须深度不可变
- **引用能力（Reference Capabilities）** — 类型编码了允许的操作（`mut`、`imm`、`iso`、`tmp`、`pau`、`var`、`cown`）
- **零拷贝区域转移** — 通过桥接对象在线程间移动整个区域
- **深度冻结** — 将关闭的区域转换为不可变的、可自由共享的对象
- **Cown（并发所有者）** — 受 Verona 启发的获取/释放协议，用于协调并发访问
- **LLVM 后端** — 通过 LLVM 编译为原生代码

### 内存安全保证

| 保证 | 机制 |
|---|---|
| 无悬挂引用 | 引用绑定到区域生命周期 |
| 无双重释放 | 区域释放需要唯一的外部桥接引用 |
| 无释放后使用 | 区域隔离阻止外部别名 |
| 无缓冲区溢出 | 编译期和运行时检查 |
| 无未初始化读取 | 所有对象均被初始化 |

## 示例

```gura
fn main() {
    print("Hello, Gura!")
}
```

```gura
fn main() {
    let region = new iso Point(10, 20)
    let value = region.enter(|r| {
        r.x + r.y
    })
    print(value)  // 30
}
```

更多示例见 [`examples/`](examples/) 目录。

## 构建

### 前置条件

- 支持 C++23 的编译器（GCC 13+、Clang 16+）
- CMake 3.24+
- Conan 2.x
- LLVM 开发库

### 构建步骤

```bash
# 安装依赖
conan install . --build=missing -s compiler.cppstd=23

# 配置并构建
cmake --preset default
cmake --build --preset default
```

### 运行测试

```bash
ctest --preset default
```

## 语言设计

### 引用能力

| 能力 | 含义 |
|---|---|
| `mut T` | 对活跃区域对象的可变引用 |
| `tmp T` | 限于当前块的临时引用 |
| `iso T` | 对关闭区域桥接的唯一外部引用 |
| `imm T` | 深度不可变，可跨线程自由共享 |
| `pau T` | 对暂停区域对象的只读视图 |
| `var T` | 支持强更新的局部存储槽 |
| `cown T` | 具有获取/释放协议的并发所有者 |

### 区域生命周期

```
new iso → 活跃（可变） → 暂停（只读） → 关闭（隔离）
                                              │
                                ┌─────────────┼─────────────┐
                                │             │             │
                              freeze        merge        spawn
                                │             │             │
                              imm T       活跃区域 R     另一个线程
                            (可共享)     (可变)
```
